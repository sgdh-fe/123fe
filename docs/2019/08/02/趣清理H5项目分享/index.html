2.1 关键技术

2.1.1 Redux

为什么使用 Redux:

  1. 单一数据源：管理数据成本降低（多人开发尤为重要);
  2. 因为接口复杂，通过 action 封装接口，降低接口调用成本;(调用方不用关心接口)
  3. action 将接口数据和本地数据打通;
  4. 技术储备

使用 Redux 遇到两个问题

  1. 异步化的action
  2. 订阅函数被频繁触发;

4.总结

  • 原则一：保证开发流程通畅;
  • 原则二：保证数据流程通畅;

保证开发流程通畅方法：

  • 明确开发流程：比如开发环境和调试环境是什么，如何进行联调，如何部署代码等等;

保证数据流通畅方法：

  • 梳理功能模块和通信接口，确定接口各个字段、类型、请求值、返回值，确定接口成功、失败等情况;
  • 通过Moc值模拟接口数据;
